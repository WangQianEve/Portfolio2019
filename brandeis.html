Design Code About Resume
Time
Feb. 2019 - Jul. 2019 (5 months)
Team
Ally Liu / Qian Wang / Rudy Iyer / Won-Woo Chung
My Role
UX Designer
The Brandeis Project is a collaboration between CMU CHIMPS Lab and DARPA. Prof. Jason Hong, lead of CHIMPS Lab, is currently talking with Google for future collaboration.
Problem
How might we help users protect their mobile privacy easily in varous context and satisfy both individual and organizational need?
Current mobile systems let user control App’s access to sensors/data at certain level. However, this is not enough. Based our research, many application request more data than they need, or even leak users’ data to other companies. This can endanger users’ property, or even life for those works in special industries (e.g., military).
Solution
A privacy-enhanced Android system that helps you control and understand your privacy settings.
How Does It Work?
Touch Point 1 - Privacy Manager
Homepage

Privacy Manager is where users can manage their privacy settings. From the homepage, users can access their global settings, app settings, and privacy profiles.

Privacy Overview
In the privacy overview section, user can get a brief view of how their data are currently being accessed, and how much are atypical.

Recommended Setting
The system generates some recommendations so that user can have a better privacy setting easily.

Global Setting

“What-Why-Where” setting
Our privacy-enhanced Android can get purpose of a data request from developers, and can trace back which specific library used the data. So we used a “what-why-where” information hierarchy, which includes 3 levels of information: “what data is being requested”, “why it’s requested”, and “where is the data used”. They are shown in a progressive disclosure fashion in both global settings and app settings.

Privacy Mode

Organizations sometimes need to prevent data leaks from its employees or visitors. For instance, DARPA’s employees sometimes need to protect their location from anyone when they are on a special mission. So we designed privacy mode, which contains a set of configuration rules that user cannot change.

Touch Point 2 - Quick Setting
Quick Setting

In quick setting view, PE-Android support more privacy-related items. This can help those who are in a sensitive situation and want to quickly cut off certain data access temporally.

Here I also showed what it looks like when current privacy mode locked certain item.

Touch Point 3 - Install Time
App Settings

Right after users installed an App, the App setting page will be prompted. By default, all requests will be set based on your usual settings, if not available, it’ll be set based on most PE-Android users’ setting.

The uncommon requests are put upfront so that if users are too busy, they can configure those first.

Here I also showed what it looks like when current privacy mode make certain settings temporally invalid.

Touch Point 4 - App Running
Run-time Message

When user is using the App, a pop-up will appear if the requested permission is set to “ask”. User can view the detailed explanation why the App need the data by clicking “Show More”. Checking “Never Ask Again” will change the App privacy setting.

Touch Point 5 - Need Exception
Notification Center

Sometimes, user need an exception for the privacy setting to use certain function. They can view the most recently blocked request in notification center, then allow once or change the setting in the privacy manager.

Main Challenges
Complex System
Multiple touchpoints make the system really complex. We need to stand in users’ shoes to simplify the logic and use design to express it clearly. Our final solution is:
1. Privacy mode has the highest control, quick setting comes next, then global setting and app setting.
2. Between global settings and app settings, the latest change is effective.
Various User Types
As a mobile OS, we have various target user types. Some of them wants extremely fine control while others may not even want to configure anything. At the end, we used progressive disclosure to serve various types of users.
Apply Existing Design Style
Our work is based on Android system. Considering both user habit and development constraints, our design style need to be as close as the current Android. However, current styles is not always enough for our features. When and how to innovate while staying close the design style is a challenge.
Research Methods
Card Sorting
We used card sorting to figure out the right information hierarchy. The result is our “what-why-where” structure.
A/B Testing
With a complicated system and various user types, we often come up with various potential designs. A/B testing helped us identify the strength and weakness of each, and inspired new ideas.
Cognitive Walkthrough
The more complex the system is, the more user testing and walkthrough it needs. We’ve been continuously doing cognitive walkthrough with users from different backgrounds during this project span to avoid missing edge cases.
Reflection

I learned a lot from this project, especially designing for a complex system, and applying existing design styles.

  • The key for designing a complex system is to simplify it through understanding the data and logic, combining an understanding of the user. We spent weeks discussing the controlling logic and the data, conducting user research to understand users' mental model before settling down on a design.
  • I also got familiar with Material Design through this project. Applying a design style is not only about imitation, it's also about innovation. How to use the existing style to create something that fits your concept is the challenge. We often refer back to the material design website and the current Android system to seek inspiration. Android developer documentation is also crucial for us to figure out what kind of modification/customization is feasible.

This project has not ended, we are in the process of talking to more sponsors and partners. In the future, we hope to keep refining the product till it can satisfy most users and more institutional needs. One direction I'm particularly interested in is to provide smart recommendations for privacy settings so that user don't have to dive into the complex system.

I hope one day, everyone is able to protect and manage their privacy easily and wisely.
